Specifications

Intro

In SySCoRe, we consider formal specification written using syntactically co-safe linear temporal logic (scLTL), which consists of atomic proposition (AP) that are either true or false. To define the specification, you can either use an scLTL formula or directly input a deterministic finite-state automaton (DFA).
To translate an scLTL specification, we use the tool LTL2BA [4], which can construct the Büchi automaton for an associated general LTL specification. Additionally, we check whether the given formula is written using scLTL (instead of full LTL) and then (if possible) rewrite the Büchi automaton to a DFA. This step is based on powerset conversion that is used to convert a nondeterministic finite-state automaton to a DFA. The complete translation from an scLTL specification to a DFA is implemented in the function TranslateSpec.

TranslateSpec

This function translates an scLTL specification to a deterministic finite-state automaton (DFA). The DFA consists of states (DFA.S), initial states (DFA.S0), final states (DFA.F), actions that activate the transitions (DFA.actions), transitions (DFA.trans) and a sink state (DFA.sink).
The function is defined in TranslateSpec.
