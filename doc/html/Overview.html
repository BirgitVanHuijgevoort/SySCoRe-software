Control synthesis for stochastic continuous state models

Intro to an overview of the SySCoRe toolbox

The main purpose of SySCoRe is to perform the complete control synthesis procedure in abstraction-based temporal logic control. It is applicable to discrete-time models, with a possibly unbounded stochastic disturbance and synthesizes a controller for syntactically co-safe linear temporal logic specifications that may be unbounded in time. The computational approach is based on the theory on approximate simulation relations [1], the coupling between models [1,2] and robust dynamic programming mappings [3].
The abstraction-based temporal logic control implemented in SySCoRe has six main steps, namely (1) translating the specification to an automaton, (2) constructing a (reduced) finite-state abstraction, (3) quantifying the similarity, (4) synthesizing a controller, (5) control refinement and (6) deployment. In SySCoRe these steps are implemented using the following functions:

0) Modelling

Define the model of the system using the classes in Models. Besides specifying the model dynamics, you also have to define the state space, input space and the regions for labeling the output space and add them to the class descriptions.

1) Specification

In SySCoRe, we consider formal specification written using syntactically co-safe linear temporal logic (scLTL). To define the specification, you can either use an scLTL formula or directly input a deterministic finite-state automaton (DFA). The complete translation from an scLTL specification to a DFA is implemented in the function TranslateSpec and is explained in Specifications.

2) Abstractions

SySCoRe includes two possible abstraction methods,namely model-order reduction for continuous-state LTI systems and a finite-state abstraction for continuous-state systems. Details can be found in Abstractions.

3) Similarity quantification

To quantify the similarity between two models, that is a model and its abstraction (either reduced-order or finite-state) we use an -stochastic simulation relation as in Definition 4 in [2]. Here, ϵ and δ represent the output and probability deviations respectively. This simulation relation allows us to consider scLTL specifications with unbounded properties [2]. More details can be found in Similarity.

4) Controller synthesis

We synthesize a robust (finite-state) controller based on the dynamic programming approach described in [3], which is robust in the sense that it takes the deviation bounds ϵ and δ into account to compute a lowerbound on the actual satisfaction probability. Furthermore, in [3, Theorem 4] they prove that the resulting control policy synthesized for the abstract model can always be refined to a control policy for the actual model. Details can be found in Controllers.

5) Control refinement

To refine an abstract finite-state controller to a controller C that can be implemented on the original continuous-state system we use one or multiple interface functions. In SySCoRe, control refinement is included in the class RefineController.This class does not only refine the finite-state input to the actual input, but it also determines the state of the finite-state model based on the state of the original model.

6) Implementation

The final step is to implement the controller on the model and perform a simulation using ImplementController.
